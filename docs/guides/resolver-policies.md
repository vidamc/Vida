# Политики доступа резолвера

Моды из `mods/` проходят [резолвер зависимостей](../modules/resolver.md) до загрузки. Помимо SemVer-ограничений в `vida.mod.json`, **окружение запуска** (лаунчер, серверный админ, тесты) может запретить выбор отдельных модов по политике безопасности или конфигурации.

## `accessDeniedIds` в `ResolverOptions`

Через неизменяемые опции резолвера задаётся множество **идентификаторов модов** (`Provider.id`), которые **никогда не попадут** в итоговую `Resolution`:

```java
ResolverOptions opt = ResolverOptions.DEFAULTS.withAccessDenied("undesired-mod");
Result<Resolution, ResolverError> r = Resolver.resolve(roots, universe, opt);
```

### Отличие от `excludes`

| Механизм | Поведение |
|----------|-----------|
| `excludes` | Мягкий отказ: если id только в опциональных зависимостях — может «пропасть» из решения; если **required** — ошибка `Excluded`. |
| `accessDeniedIds` | Любой провайдер с этим **id** отбрасывается при выборе. Если единственный подходящий кандидат запрещён политикой — `AccessPolicyDenied`. |

Используйте **deny**, когда мод должен считаться «токсичным» или отключённым политикой независимо от графа зависимостей; **exclude** — когда нужно исключить опциональную ветку без жёсткого запрета на уровне id.

## Ошибка `ResolverError.AccessPolicyDenied`

Возвращается в двух типичных случаях:

1. **Корневой мод** из `roots` совпадает с запрещённым id (fail-fast при старте резолва).
2. **Жёсткая зависимость** требует id, для которого все кандидаты в `Universe` запрещены политикой (или единственный кандидат — запрещён).

Для диагностики смотрите поля записи: `id`, `requesters`, `detail`.

## Связь с загрузчиком

`BootSequence` передаёт в резолвер `ResolverOptions.DEFAULTS.withAccessDenied(bootOptions.accessDeniedIds())`.

### JVM и агент

| Источник | Формат |
|----------|--------|
| Системное свойство `-Dvida.accessDenied=id1,id2` | запятые, id как в `vida.mod.json` |
| Аргумент агента `-javaagent:vida-loader.jar=accessDenied=one` | один id за раз: строка агента режется по **запятым** между парами `key=value`, поэтому список из нескольких id сюда **не помещается** |

Значение из аргумента агента перекрывает свойство `vida.accessDenied` (как и остальные ключи premain). Для **нескольких** запрещённых id используйте `-Dvida.accessDenied=a,b,c` или программный `BootOptions.builder().accessDenied(Set.of(...))`.
