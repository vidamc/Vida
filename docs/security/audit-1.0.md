# Security audit — Vida 1.0.0

**Дата отчёта:** 2026-04-22  
**Объём:** репозиторий Vida (загрузчик модов Minecraft 1.21.1), монорепо модулей и инсталлятор.  
**Метод:** статический обзор архитектуры, точек доверия, цепочек поставки и автоматизированных проверок (CI), без внешнего пентеста по развёрнутой инфраструктуре.

## Итоговая оценка

Проект следует современным практикам для OSS: нет хранения секретов в репозитории, артефакты релиза сопровождаются хэшами и **Sigstore (keyless OIDC)**; зависимости тянутся преимущественно с **Maven Central**; для кода включены **CodeQL** и **OpenSSF Scorecard** (см. `.github/workflows/`). Остаточные риски связаны с **неизбежным доверием к байт-коду модов** и к **агентной модели** (`Instrumentation`), что свойственно любой системе загрузчиков для модифицируемой игры.

## Доверенные границы

| Зона | Риск | Митигация |
|------|------|-----------|
| Код модов | Произвольный bytecode | Рекомендуемая изоляция через слои класслоадеров / политики (roadmap); в 1.0 авторы модов считаются доверенными коду игры. |
| Загрузчик / Vifada | Трансформация классов | Морфы ограничены декларативной моделью; ошибки трансформации должны быть fail-closed в релизных путях (тесты + CI). |
| Инсталлятор | Запись в каталог игрока | CLI dry-run, валидация путей; пользователь явно задаёт `--dir`. |
| Сеть | Загрузка зависимостей | Gradle + HTTPS к репозиториям; релизы через GitHub. |

## Целостность и цепочка поставки

- **Checksums:** `SHA256SUMS` / `SHA512SUMS` на релизных артефактах.
- **Подпись:** `cosign sign-blob` (Sigstore), вложения `*.sig` / `*.pem`, build provenance через GitHub attestations (`release.yml`).
- **SBOM:** CycloneDX JSON для содержимого `dist/` (генерация **Syft** в CI) — состав JAR для анализа уязвимостей и лицензий.
- **Зависимости:** централизованный каталог `gradle/libs.versions.toml`, Dependabot; рекомендуется периодический `gradle dependencyUpdates` вручную.

## Рекомендации на последующие минорные релизы

1. **Supply chain:** при публикации в Maven — требовать **sigstore/cosign** или GPG подпись координат (уже есть для GitHub Releases).
2. **Мод-слой:** документировать для мод-авторов минимальный набор привилегий (без рефлексии на сокетах/ФС без явного разрешения) — политика остаётся на стороне установки.
3. **Инсталлятор:** при добавлении новых launcher handlers — сохранять контракт «не перезаписывать чужие профили без подтверждения».

## Ограничения отчёта

Этот документ **не заменяет** независимый внешний аудит с оплачиваемым ретейнером; он фиксирует состояние кодовой базы и пайплайна на момент тега **1.0.0** и служит отправной точкой для будущих ревью.
