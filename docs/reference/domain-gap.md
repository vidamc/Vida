# Доменные пробелы API относительно ТЗ «все аспекты игры»

ТЗ ([vida.md](../../vida.md), ч.3) формулирует идеал «полного контроля». Ниже — явная классификация доменов Minecraft **1.21.1**, чтобы мододелы и планировщики не искали несуществующие пакеты.

| Домен | Статус в Vida | Комментарий |
|-------|---------------|-------------|
| Блоки, предметы, компоненты данных | Закрыто (`vida-bloque`, `vida-objeto`) | См. модули |
| Сущности | Закрыто (`vida-entidad`) | 2.0+ Stable |
| Мир, чанки, биомы, тики мира | Закрыто (`vida-mundo`) | Latidos мира |
| Сеть | Закрыто (`vida-red`, Tejido) | Включая фрагментацию крупных payload |
| Рендер декларативный / HUD | Закрыто (`vida-render`) | HUD через `LatidoRenderHud`; сложный Screen UI — клиентский код или расширения render |
| Ресурсы и datapack | Закрыто (`vida-fuente`) | Data-driven + `LatidoFuenteRecargada` |
| Конфигурации | Закрыто (`Ajustes`, перезагрузка, синхронизация) | См. [ajustes.md](../guides/ajustes.md) |
| Звуки, частицы, кастомные статистики | **В плане roadmap** | Нет отдельного `vida-sonido` / `vida-particula`; использовать vanilla + Vifada до появления модулей |
| Прогресс / advancements / квесты высокого уровня | **В плане** | Через Fuente / игровые JSON, не отдельный модуль |
| Полный UI Screen API | **Вне минимального контура** | Экраны остаются на Mojmap-классах + Vifada; `vida-render` не дублирует виджетную библиотеку |

Новые стабильные модули добавляются только с SemVer-мажором при breaking changes к существующим `@Stable` типам.
