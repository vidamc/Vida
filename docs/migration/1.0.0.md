# Миграция на Vida 1.0.0

Релиз **1.0.0** фиксирует публичный API ключевых модулей платформы и обещание [SemVer 2.0.0](https://semver.org/lang/ru/) для потребителей на линии `1.x`.

## Что изменилось по смыслу

- Модули **`vida-base`**, **`vida-bloque`**, **`vida-objeto`**, **`vida-susurro`**, **`vida-puertas`** и публичный API Gradle-плагина **`dev.vida.gradle`** переведены с `@ApiStatus.Preview` на **`@ApiStatus.Stable`**. Breaking changes в этих пакетах допускаются только в **мажорной** версии платформы (`2.0.0` и выше).
- Примеры в документации с полным `package dev.vida....` проверяются сборкой задачей `./gradlew vidaDocTest` (компиляция извлечённых фрагментов).
- Релизные артефакты по-прежнему подписываются **Sigstore (keyless)**; к релизу добавляется **CycloneDX SBOM** (генерация из собранных JAR в CI).

## Если вы начинали с 0.x

1. Обновите зависимости в `vida.mod.json` на `">=1.0.0 <2.0.0"` там, где требовалась стабильная платформа.
2. Просмотрите `CHANGELOG.md` для вашей предыдущей версии — всё, что помечено как breaking для preview-модулей (`vifada`, `loader`, …), могло затронуть только эти теги; **стабильные** пакеты из списка выше в 1.0 не должны требовать миграции, если вы не опирались на **внутренние** пакеты (`@Internal`, `*.internal.*`).
3. Gradle: обновите строку плагина `plugins { id("dev.vida.mod") version "1.0.0" }` (или диапазон по SemVer).

## Что остаётся в Preview

`vifada`, `loader`, `installer`, `entidad`, `mundo`, `render`, `red`, `vigia` и др. сохраняют `@ApiStatus.Preview` до выполнения критериев в [api-stability.md](../reference/api-stability.md). Следите за тегами в `CHANGELOG` и roadmap.

## SemVer для Vida

- **MAJOR** — ломающие изменения в типах с `@Stable`.
- **MINOR** — новые API, поведение по умолчанию совместимо.
- **PATCH** — исправления ошибок без изменения контракта `@Stable`.

Исключения и оговорки для `record` / `sealed` описаны в [api-stability.md](../reference/api-stability.md).
