# Migración 1.x LTS → 2.0 «Масштаб»

## SemVer

2.0 es un major: pueden existir cambios en API marcados como `@Stable` en 1.x solo si llevaron `@Deprecated(forRemoval=true)` ≥12 meses y nota en `CHANGELOG.md`, salvo excepciones documentadas.

## LTS 1.x

La línea **1.0.x** permanece como **LTS**: durante **24 meses** desde el último parche publicado en esa línea se corrigen vulnerabilidades de seguridad y fallos críticos de arranque/resolución según el proceso descrito en `CHANGELOG.md` y los anuncios de release.

## Contenido data-driven

- Se amplía `custom["vida:dataDriven"]` con tablas de loot bajo `{datapackRoot}/loot_tables/` (véase `docs/reference/manifest-schema.md`).
- Un mod solo con JSON en datapack se construye según `vida:dataDriven` (véase el esquema del manifiesto); un ejemplo aislado en el árbol `mods/` de este repositorio ya no se mantiene.

## Vifada 2

- `@VifadaMulti`, `@VifadaLocal`, `@VifadaRedirect` están implementados en el transformer.
- Paquete `dev.vida.vifada.preview` eliminado: usar anotaciones canónicas en `dev.vida.vifada`.

## Hot reload (desarrollo)

- Ver `docs/guides/hot-reload.md`. La opción Gradle `vida.run { hotReload.set(true) }` activa watcher y reset controlado de `CatalogoManejador`.

## Telemetría

- Ver `docs/security/telemetry-v1.md`.

## Instalador

- Caché offline opcional: variable de entorno `VIDA_INSTALL_CACHE` o `-Dvida.install.cache=<dir>` para reutilizar `loader.jar` extraído.

## API `vida-mundo` (`@Stable`)

- Contrato documentado en [`docs/modules/mundo.md`](../modules/mundo.md): `Mundo` (defaults `limitesVerticales`, `enRangoDeAltura`, ciclo día/noche), `Coordenada`, `ChunkCoordenada`, `RegionCoordenada`, `LimitesVerticales` (perfiles Vanilla 1.21.x), `Dimension.limitesVerticalesPredeterminados()`, `MundoEstatico`, `Bioma`, `LatidosMundo` (incl. `ChunkDescargado`).
- Ejemplos con `package dev.vida....` se compilan en `./gradlew vidaDocTest`.
