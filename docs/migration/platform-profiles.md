# Миграция: профили платформы

С **Vida 1.x** вводится опциональный дескриптор **профиля платформы** (`platform-profiles/.../profile.json`), упаковываемый в jar загрузчика под `META-INF/vida/platform-profiles/`.

## Для авторов модов

- **Без изменений**, если вы не полагаетесь на точное совпадение дропа MC: не задавайте профиль и не добавляйте `custom.vida.platformProfileIds`.
- Если вы коразрабатываете морфы под конкретные дропы и включаете профиль лаунчера, добавьте в `vida.mod.json`:

```json
"custom": {
  "vida": {
    "platformProfileIds": ["legacy-121/1.21.7"]
  }
}
```

Тогда при несовпадении активного профиля Vifada-морфы из вашего JAR не регистрируются (раньше это могло заканчиваться обфусцированными `NoSuchMethodError` глубже по стеку).

## Для сборки (`vida` Gradle DSL)

Рекомендуется задать `vida.minecraft.version` и либо явный `vida.minecraft.platformProfile`, либо полагаться на convention `legacy-121/<version>` для строк вида `1.21.*`.
